Documentation

"MyFlix" is a web application that provides users with access to information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.
The server-side of the application (server, business logic, business layers) consists of a REST API and architected database built using JavaScript, Node.js, Express, and MongoDB.

Movie requests

Request URL HTTP Method Request body data format Response body data format
Get a list of all movies /movies GET None JSON object holding data about all movies
Get data about a single movie by title /movies/[Title] GET None JSON object holding data about a single movie, containing title, description, genre, director, imageURL, featured or not.
Example:
{
id: 1,
title: "Bohemian Rhapsody",
year: 2018,
description: "Description about Bohemian Rhapsody",
genre: {
name: "biographical drama",
description: "Description of biographical drama"
},
director: {
name: "Bryan Singer",
bio: "Bio of Bryan Singer",
birth: "1965",
death: ""
},
imageURL: "",
featured: ""
}
Get data about a genre by name /genres/[Name] GET None A JSON object holding data about a movie genre containing name and description.
Example:
{
name: "comedy drama",
description: "Description of comedy drama"
}
Get data about a director by name /directors/[Name] GET None JSON object holding data about a director, containing name, bio, date of birth and death.
Example:
{
name: "Bryan Singer",
bio: "Bio of Bryan Singer",
birth: "1965",
death: ""
}

User Requests

Request URL HTTP Method Request body data format Response body data format
Add a new user /users POST JSON object holding data about a user, structured like:
{
username: "Polly Pocket",
password: "",
email: "polly@pocket.com",
birthday: ""
} 		JSON object holding data about the user that was added, including an ID.
Example
{
id: 1,
username: "Polly Pocket",
password: "",
email: "polly@pocket.com",
birthday: ""
}
Update user information by username /users/[Username] PUT JSON object holding data to be updated, structured like:
{
id: 1,
username: "Polly Pocket",
password: "",
email: "polly@pocket.com",
birthday: ""
} 		A text message indicating the user's information was updated
Add a movie to list of favorites by ID /users/[Username]/movies/[MovieID] POST None A text message indicating the movie was added to the list of favorites
Remove a movie from the list of favorites by movieID /users/[Username]/movies/[MovieID] DELETE None A text message indicating whether the movie was successfully removed.
Remove user account by userID /users/[UserID] DELETE None A text message indicating whether the user account was successfully deleted.
